1 /++
2 This module is implementing the Expected idiom.
3 
4 See the [Andrei Alexandrescu’s talk (Systematic Error Handling in C++](http://channel9.msdn.com/Shows/Going+Deep/C-and-Beyond-2012-Andrei-Alexandrescu-Systematic-Error-Handling-in-C)
5 and [its slides](https://skydrive.live.com/?cid=f1b8ff18a2aec5c5&id=F1B8FF18A2AEC5C5!1158).
6 
7 Or more recent ["Expect the Expected"](https://www.youtube.com/watch?v=nVzgkepAg5Y) by Andrei Alexandrescu for further background.
8 
9 It is also inspired by C++'s proposed [std::expected](https://wg21.link/p0323) and [Rust's](https://www.rust-lang.org/) [Result](https://doc.rust-lang.org/std/result/).
10 
11 Similar work is [expectations](http://code.dlang.org/packages/expectations) by Paul Backus.
12 
13 ## Main features
14 
15 $(LIST
16     * lightweight, no other external dependencies
17     * works with `pure`, `@safe`, `@nogc`, `nothrow`, and `immutable`
18     * provides methods: `expected`, `unexpected`, `andThen`, `orElse`, `map`, `mapError`, `mapOrElse`
19     * type inference for ease of use with `expected` and `unexpected`
20     * allows to use same types for `T` and `E`
21     * allows to define $(LREF Expected) without value (`void` for `T`)
22     * provides facility to change the $(LREF Expected) behavior by custom `Hook` implementation using the Design by introspection paradigm.
23     * can enforce result check (with a cost)
24 )
25 
26 ## Description
27 
28 Actual $(LREF Expected) type is defined as $(D Expected!(T, E, Hook)), where:
29 
30 $(LIST
31 * `T` defines type of the success value
32 * `E` defines type of the error
33 * `Hook` defines behavior of the $(LREF Expected)
34 )
35 
36 Default type for error is `string`, i.e. `Expected!int` is the same as `Expected!(int, string)`.
37 
38 $(LREF Abort) is used as a default hook.
39 
40 ### Hooks
41 
42 $(LREF Expected) has customizable behavior with the help of a third type parameter,
43 `Hook`. Depending on what methods `Hook` defines, core operations on the
44 $(LREF Expected) may be verified or completely redefined.
45 If `Hook` defines no method at all and carries no state, there is no change in
46 default behavior.
47 This module provides a few predefined hooks (below) that add useful behavior to
48 $(LREF Expected):
49 
50 $(BOOKTABLE ,
51     $(TR $(TD $(LREF Abort)) $(TD
52         fails every incorrect operation with a call to `assert(0)`.
53         It is the default third parameter, i.e. $(D Expected!short) is the same as
54         $(D Expected!(short, string, Abort)).
55     ))
56     $(TR $(TD $(LREF Throw)) $(TD
57         fails every incorrect operation by throwing an exception.
58     ))
59 )
60 
61 The hook's members are looked up statically in a Design by Introspection manner
62 and are all optional. The table below illustrates the members that a hook type
63 may define and their influence over the behavior of the `Checked` type using it.
64 In the table, `hook` is an alias for `Hook` if the type `Hook` does not
65 introduce any state, or an object of type `Hook` otherwise.
66 
67 $(TABLE_ROWS
68     * + Hook member
69       + Semantics in Expected!(T, E, Hook)
70     * - `enableDefaultConstructor`
71       - If defined, $(LREF Expected) would have enabled or disabled default constructor
72         based on it's `bool` value. Default constructor is disabled by default.
73         `opAssign` for value and error types is generated if default constructor is enabled.
74     * - `enableCopyConstructor`
75       - If defined, $(LREF Expected) would have enabled or disabled copy constructor based
76         on it's `bool` value. It is enabled by default. When disabled, it enables automatic
77         check if the result was checked either for value or error.
78         When not checked it calls $(D hook.onUnchecked) if provided.
79 
80         $(NOTE WARNING: As currently it's not possible to change internal state of `const`
81         or `immutable` object, automatic checking would't work on these. Hopefully with
82         `__mutable` proposal..)
83     * - `onAccessEmptyValue`
84       - If value is accessed on unitialized $(LREF Expected) or $(LREF Expected) with error
85         value, $(D hook.onAccessEmptyValue!E(err)) is called. If hook doesn't implement the
86         handler, `T.init` is returned.
87     * - `onAccessEmptyError`
88       - If error is accessed on unitialized $(LREF Expected) or $(LREF Expected) with value,
89         $(D hook.onAccessEmptyError()) is called. If hook doesn't implement the handler,
90         `E.init` is returned.
91     * - `onUnchecked`
92       - If the result of $(LREF Expected) isn't checked, $(D hook.onUnchecked()) is called to
93         handle the error. If hook doesn't implement the handler, assert is thrown.
94         $(NOTE Note that `hook.enableCopyConstructor` must be false for checks to work.)
95 )

106 /// Basic usage
107 @safe unittest
108 {
109 	auto foo(int i) {
110 		if (i == 0) return unexpected!int("oops");
111 		return expected(42 / i);
112 	}
113 
114 	auto bar(int i) {
115 		if (i == 0) throw new Exception("err");
116 		return i-1;
117 	}
118 
119 	// basic checks
120 	assert(foo(2));
121 	assert(foo(2).hasValue);
122 	assert(!foo(2).hasError);
123 	assert(foo(2).value == 21);
124 
125 	assert(!foo(0));
126 	assert(!foo(0).hasValue);
127 	assert(foo(0).hasError);
128 	assert(foo(0).error == "oops");
129 
130 	// void result
131 	assert(expected()); // no error -> success
132 	assert(!expected().hasError);
133 	// assert(unexpected("foo").hasValue); // doesn't have hasValue and value properties
134 
135 	// expected from throwing function
136 	assert(expected!bar(1) == 0);
137 	assert(expected!bar(0).error.msg == "err");
138 
139 	// orElse
140 	assert(foo(2).orElse!(() => 0) == 21);
141 	assert(foo(0).orElse(100) == 100);
142 
143 	// andThen
144 	assert(foo(2).andThen(foo(6)) == 7);
145 	assert(foo(0).andThen(foo(6)).error == "oops");
146 
147 	// map
148 	assert(foo(2).map!(a => a*2).map!(a => a - 2) == 40);
149 	assert(foo(0).map!(a => a*2).map!(a => a - 2).error == "oops");
150 
151 	// mapError
152 	assert(foo(0).mapError!(e => "OOPS").error == "OOPS");
153 	assert(foo(2).mapError!(e => "OOPS") == 21);
154 
155 	// mapOrElse
156 	assert(foo(2).mapOrElse!(v => v*2, e => 0) == 42);
157 	assert(foo(0).mapOrElse!(v => v*2, e => 0) == 0);
158 }

168 /++
169 	`Expected!(T, E)` is a type that represents either success or failure.
170 
171 	Type `T` is used for success value.
172 	If `T` is `void`, then $(LREF Expected) can only hold error value and is considered a success when there is no error value.
173 
174 	Type `E` is used for error value.
175 	The default type for the error value is `string`.
176 
177 	Default behavior of $(LREF Expected) can be modified by the `Hook` template parameter.
178 
179 	Params:
180 		T    = represents the expected value
181 		E    = represents the reason explaining why it doesn’t contains avalue of type T, that is the unexpected value.
182 		Hook = defines the $(LREF Expected) type behavior

565 /// Template to determine if provided Hook enables default constructor for $(LREF Expected)
566 template isDefaultConstructorEnabled(Hook)
567 {
568 	static if (__traits(hasMember, Hook, "enableDefaultConstructor"))
569 	{
570 		static assert(
571 			is(typeof(__traits(getMember, Hook, "enableDefaultConstructor")) : bool),
572 			"Hook's enableDefaultConstructor is expected to be of type bool"
573 		);
574 		static if (__traits(getMember, Hook, "enableDefaultConstructor")) enum isDefaultConstructorEnabled = true;
575 		else enum isDefaultConstructorEnabled = false;
576 	}
577 	else enum isDefaultConstructorEnabled = false;
578 }

584 	struct Bar { static immutable bool enableDefaultConstructor = true; }
585 	static assert(!isDefaultConstructorEnabled!Foo);

637 /++ Template to determine if hook enables or disables copy constructor.
638 
639 	It is enabled by default.
640 
641 	See $(LREF hasOnUnchecked) handler, which can be used in combination with disabled
642 	copy constructor to enforce result check.
643 
644 	$(WARNING If copy constructor is disabled, it severely limits function chaining
645 	as $(LREF Expected) needs to be passed as rvalue in that case.)
646 +/
647 template isCopyConstructorEnabled(Hook)
648 {
649 	static if (__traits(hasMember, Hook, "enableCopyConstructor"))
650 	{
651 		static assert(
